Future of Web Design

This project is a single-page website dedicated to the impact of SVG in future web design.
  • SVG: What is it?

    Scalable Vector Graphics(SVG) is an XML-based vector image format that is open-source and supported by all modern browsers. SVG is developed and maintained by the W3C SVG Working Group. The most recent version of the W3C Recommended full specification is SVG 1.1 2nd edition

    SVG is a platform for two-dimiesional graphics. It has two parts: an XML-based file format and a programming API for graphical applications. SVG is used in many business areas including web graphics, animation, user interfaces, graphics interchange, mobile applications and high-quality design. It is even popular with crafters when creating high-resolution designs that will be cut out for craft and art projects. SVG is open source and builds upon many other standards including XML, JPEG and PNG for image formats, DOM (Document Object Model) for scripting and interactivity, SML for animation and CSS for styling. Javascript can be used to manipulate both the objects and the DOM. SVG can now be read by most browsers, and since it is XML-based, it is written in text, which means SVG code can be read by both humans and machines.

  • Why use SVG?

    SVG is XML, making it easy to understand and allowing for manual editing using a text editor. SVG offers a way to do full resolution graphical elements, no matter what size screen, what zoom level, or what resolution your user's device has.

    Examples of some practical applications for SVG are Graphs, Road Maps, Complex UI Elements, Logos, and Simple Games.

    Resolution Independence:

    SVG looks crisp on any screen resolution and offers responsive images for responsive web design. SVG can also be used on any background color and the color of the image will be true. There is no loss of clarity when SVG images are scaled up or down. Also SVG images are vector graphics so there is no need to create separate images for high-density (retina) screens.

    Animation:

    SVG animations can be customized and controlled inside the SVG code easily. SVG animations can be reused and gzipped (compressed), making their file sizes smaller, especially when compared to GIFs. This improves page load times and performance. The file sizes are smaller because SVG code only lists the lines, curves and shapes required for the image.

    Accessibility:

    SVG graphics and animations are accessible to assistive technologies(AT), such as screen readers and speech recognition tools. SVG code can also be customized without resorting to hacking the source code. This is beneficial to people with visually-triggered vestibular disorders, motion on a webpage makes them feel ill.

    Ease of Use:

    SVG images can be created in a regular text editor such as Windows Notepad, although you would need an SVG viewer to see the resulting image.

  • Tips for Accessibility

    The best way to make SVG accessible to assistive technologies like screen readers and speech recognition tools is to put it directly in your HTML.

    Provide a title. The SVG <title> element provides a human readable name for the SVG content.
    Provide a description. The SVG <desc> element provides more detailed information about the SVG than the title element allows.
    Use Text. The SVG <text> element makes text in a graphical content highly accessible. The content of the text element is rendered as plain text by the browser, so it's available to screen readers, and because it's SVG it can be resized porportionately with the graphical content for the benefit of anyone with less than 20/20 vision.
    Make it focusable. If the SVG contains content that is interactive, make sure it can take keyboard focus. The best way to do this is to use an element natively designed to take focus in the first place.
    Give it a role. This improves the browser odds considerably, with IE, Firefox, Chrome and Safari all exposing the role correctly.
    Create an alternative. SVG 2.0 should mostly eliminate the need for this, but an example is information presented in an SVG graph might also be provided in an HTML table, perhaps with a choice of views offered through a tabbed interface.

    The DOM API provides a further set of clear advantages for using SVG: No unnecessary HTTP requests and easy interactive scripting.
